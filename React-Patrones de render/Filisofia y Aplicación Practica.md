# Filosifía React?
- Abstracciones comunes: No especifidad, permite libertad al desarrollador
- Interoperabilidad: Permitir la migración de manera rapida y confiable, herraminetas ya adaptadas para el desarrrollo

## Cambios al core?
- Estabilidad: mantener codigo legacy durante cierto tiempo para poder permitir la adaptación del nuevo codigo
- Válvulas de escape: Geenra alertas para entender que funcionalidad se descontinuara y por cual deberiamos cambiarla.  
- Experiencias de desarrollo: Hacer de la experiencia de desarrollo algo placentero para el desarrollador

## Prioridades

- Implementación: Codigo constante(facil de actualizar) sobre codigo elegante
- Optimización para instrumentación: Usar descripción de los nombres lo mas apropiado y cortamente posible
- Dogfooding: Prioridad de los intereses de facebook, ya que en un principio React fue creado para ello.

## Trabajo de React

- Planificación: React permitira una construcción del proyecto apropiada gracias a sus herramientas, de manera descriptiva
- Configuración: Sera casi exclusivo de react, omitiendo la necesidad de hacerlo por parte de los desarrolladores.
